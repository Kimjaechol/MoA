# MoA (Multi-device Orchestration Agent) 시스템 특허명세서

## 발명의 명칭

**복잡도 적응형 AI 모델 라우팅 및 프라이버시 보호 기반의 다중 디바이스 원격 제어 시스템 및 방법**

(영문: System and Method for Complexity-Adaptive AI Model Routing and Privacy-Preserving Multi-Device Remote Control)

---

## 기술분야

본 발명은 인공지능 에이전트 시스템에 관한 것으로, 더욱 상세하게는 메시징 플랫폼을 통해 복수의 디바이스를 원격으로 제어하고, 사용자 요청의 복잡도와 민감도를 자동 분석하여 최적의 AI 모델을 선택하며, 다중 디바이스 간 장기 기억을 실시간 동기화하는 시스템 및 방법에 관한 것이다.

---

## 배경기술

### 종래 기술의 문제점

1. **단일 디바이스 한정**: 기존의 AI 어시스턴트(ChatGPT, Claude, Gemini 등)는 해당 앱이 설치된 단일 디바이스에서만 작동하며, 다른 디바이스를 원격으로 제어할 수 없다.

2. **시스템 접근 불가**: 기존 AI는 샌드박스 환경에서 실행되어 파일 시스템, 터미널, 브라우저 등 시스템 자원에 접근할 수 없다.

3. **기억의 단절**: 디바이스를 변경하면 이전 대화 맥락과 사용자 선호도가 유지되지 않아 처음부터 다시 시작해야 한다.

4. **일률적 모델 적용**: 간단한 질문이든 복잡한 분석이든 동일한 AI 모델을 사용하여 비용 효율성이 낮다.

5. **프라이버시 위험**: 민감한 개인정보(주민번호, 의료기록 등)가 클라우드 서버로 전송되어 보안 위험이 있다.

---

## 발명의 내용

### 해결하고자 하는 과제

본 발명은 상기와 같은 종래 기술의 문제점을 해결하기 위하여 안출된 것으로:

1. 메시징 플랫폼(카카오톡, 텔레그램 등)을 통해 복수의 디바이스를 안전하게 원격 제어하는 시스템을 제공한다.

2. 사용자 요청의 복잡도를 자동 분석하여 최적의 AI 모델(무료/저가/고급)을 선택하는 지능형 라우팅 시스템을 제공한다.

3. 민감한 개인정보를 자동 감지하여 로컬 SLM(Small Language Model)에서 처리함으로써 프라이버시를 보호하는 시스템을 제공한다.

4. 다중 디바이스 간 장기 기억(Long-term Memory)을 실시간 동기화하여 연속적인 사용자 경험을 제공한다.

5. 중앙 보안 레이어를 통해 위험한 명령을 사전 차단하고 사용자 확인을 요청하는 시스템을 제공한다.

### 과제의 해결 수단

본 발명에 따른 다중 디바이스 원격 제어 시스템은:

1. **메시징 컨트롤러 모듈**: 메시징 플랫폼(카카오톡, 텔레그램, 슬랙 등)에서 사용자 명령을 수신하고, 처리 결과를 반환하는 인터페이스 모듈

2. **중앙 릴레이 서버**:
   - 사용자 인증 및 디바이스 토큰 검증
   - 명령 암호화 및 보안 검사
   - 위험도 분석 및 사용자 확인 요청
   - 다중 디바이스 라우팅

3. **복잡도 분류기(Complexity Classifier)**:
   - 사용자 메시지의 복잡도를 1-5 레벨로 분석
   - 코드 작성, 전문 분야, 분석 요청 등 패턴 감지
   - 복잡도에 따른 AI 모델 티어 추천 (무료/저가/고급)

4. **프라이버시 분류기(Privacy Classifier)**:
   - 민감 정보(PII, 금융, 의료, 인증) 자동 감지
   - 민감 데이터 마스킹 처리
   - 로컬 SLM 라우팅 결정

5. **스마트 모델 라우터(Smart Model Router)**:
   - 복잡도와 프라이버시 분석 결과 통합
   - 사용자 API 키 보유 여부 확인
   - 최적 모델 자동 선택 또는 사용자 확인 요청

6. **디바이스 에이전트(Device Agent)**:
   - 사용자 디바이스에 설치되는 로컬 에이전트
   - 도구 실행기(Tool Executor)를 통한 시스템 명령 실행
   - 로컬 SLM 실행 환경
   - E2E 암호화 통신

7. **메모리 동기화 모듈**:
   - 다중 디바이스 간 장기 기억 실시간 동기화
   - 대화 맥락, 사용자 선호도, 작업 패턴 저장
   - 충돌 해결 알고리즘

### 복잡도 분류 알고리즘

```
입력: 사용자 메시지 M
출력: 복잡도 레벨 L (1-5), 추천 모델 티어 T

1. 기본 점수 S = 2

2. 패턴 분석:
   - IF M contains 코드/기술 패턴 THEN S += 2
   - IF M contains 전문분야 패턴 (법률/의료/금융) THEN S += 2
   - IF M contains 분석/비교 요청 THEN S += 1
   - IF M contains 창작 요청 THEN S += 1
   - IF M contains 복수 질문 THEN S += 1
   - IF length(M) > 500자 THEN S += 1
   - IF word_count(M) > 20 THEN S += 0.5
   - IF word_count(M) > 50 THEN S += 0.5

3. 예외 처리:
   - IF M matches 간단한_인사_패턴 AND word_count(M) < 10 THEN S = 1
   - IF word_count(M) < 5 AND NOT 코드 AND NOT 전문분야 THEN S = 1

4. 정규화: L = min(5, max(1, round(S)))

5. 티어 결정:
   - IF L <= 2 THEN T = "free"
   - ELSE IF L <= 3 THEN T = "cheap"
   - ELSE T = "premium"

6. RETURN (L, T)
```

### 프라이버시 분류 알고리즘

```
입력: 사용자 메시지 M
출력: 프라이버시 레벨 P, 로컬처리 필요여부 R

1. 민감 패턴 집합 SP = {
     critical: [주민번호, 카드번호, 비밀번호, API키, ...],
     sensitive: [계좌번호, 진단정보, 개인사진, ...],
     private: [일기, 건강기록, 위치정보, ...]
   }

2. 감지된 타입 D = {}

3. FOR EACH (level, patterns) IN SP:
     FOR EACH pattern IN patterns:
       IF pattern matches M:
         D.add((level, pattern))
         BREAK

4. 레벨 결정:
   - IF "critical" IN D THEN P = "critical"
   - ELSE IF "sensitive" IN D THEN P = "sensitive"
   - ELSE IF "private" IN D THEN P = "private"
   - ELSE P = "public"

5. 로컬 처리 결정:
   - IF P == "critical" OR P == "sensitive" THEN R = true
   - ELSE R = false

6. RETURN (P, R)
```

## 청구범위

### 독립항

**[청구항 1]**
메시징 플랫폼을 통해 복수의 사용자 디바이스를 원격 제어하는 시스템에 있어서,

상기 메시징 플랫폼으로부터 사용자 명령을 수신하는 메시징 인터페이스 모듈;

상기 사용자 명령의 복잡도를 복수의 레벨로 분류하는 복잡도 분류기;

상기 사용자 명령에 포함된 민감 정보를 감지하고 프라이버시 레벨을 결정하는 프라이버시 분류기;

상기 복잡도 레벨과 프라이버시 레벨에 기초하여 복수의 AI 모델 중 하나를 선택하는 스마트 모델 라우터;

상기 사용자의 복수 디바이스 간 장기 기억을 실시간 동기화하는 메모리 동기화 모듈; 및

상기 사용자 디바이스에 설치되어 상기 선택된 AI 모델의 응답에 따라 시스템 명령을 실행하는 디바이스 에이전트;

를 포함하는 것을 특징으로 하는 다중 디바이스 원격 제어 시스템.

**[청구항 2]**
인공지능 모델을 이용한 사용자 요청 처리 방법에 있어서,

(a) 메시징 플랫폼을 통해 사용자 명령을 수신하는 단계;

(b) 상기 사용자 명령의 복잡도를 복수의 레벨(간단, 일반, 복잡, 전문가)로 분류하는 단계;

(c) 상기 사용자 명령에 포함된 민감 정보 유형(개인식별정보, 금융정보, 의료정보, 인증정보)을 감지하는 단계;

(d) 상기 복잡도 레벨이 소정 임계치 이상이고 사용자가 고급 AI 모델의 API 키를 보유하지 않은 경우, 고급 모델 사용을 위한 확인 요청을 생성하는 단계;

## 발명의 효과

1. **원격 제어 편의성**: 카카오톡 등 일상적으로 사용하는 메시징 앱만으로 자신의 모든 디바이스를 어디서든 제어할 수 있다.

2. **비용 최적화**: 간단한 질문에는 무료 모델, 복잡한 분석에만 고급 모델을 사용하여 AI 사용 비용을 절감한다.

3. **프라이버시 보호**: 민감한 개인정보는 로컬에서만 처리되어 클라우드로 유출되지 않는다.

4. **연속적 사용자 경험**: 디바이스를 변경해도 이전 대화 맥락과 선호도가 유지되어 끊김 없는 경험을 제공한다.

5. **보안 강화**: 중앙 보안 레이어가 위험한 명령을 사전 차단하고, E2E 암호화로 통신을 보호한다.

6. **사용자 선택권 보장**: 고급 모델 사용 시 API 키 등록, 크레딧 사용, 무료 시도 중 선택할 수 있다.
